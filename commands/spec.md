Load the spec skill, then create a project spec for: $ARGUMENTS

Follow the spec skill process:
1. Gather context — read existing docs, design docs, brainstorming notes
2. Interview if needed — ask questions one at a time to fill gaps
3. Draft the spec — vision, goals, non-goals, overview, architecture, key decisions, edge cases, open questions
4. Present for review — get feedback
5. Revise — incorporate feedback
6. Save to docs/SPEC.md
7. Offer to invoke the plan skill next
